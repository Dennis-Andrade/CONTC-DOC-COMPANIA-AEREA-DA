Implementar un sistema de procesamiento de eventos asíncrono utilizando operadores funcionales y la librería Project Reactor en Spring Boot (WebFlux). Además, el estudiante deberá demostrar competencias profesionales gestionando el código fuente mediante Git y GitHub
Configuración del Repositorio (Rama: main)
Crea un proyecto Spring Boot con la dependencia Spring Reactive Web (WebFlux). Inicializa un repositorio en GitHub.
Ejemplo de commit semántico: chore: inicializa proyecto spring webflux
Actividad 1: Modelo Inmutable (Rama: feature/modelo-inmutable)
Crea una clase ReservationEvent que sea 100% inmutable. Debe contener al menos: id (String), passengerName (String), price (Double), y emails (List<String>).
Requisito técnico: Uso estricto de final, ausencia de setters, y uso de copias defensivas en el constructor y el getter de la lista de correos.
Ejemplo de commit semántico: feat: agrega entidad inmutable ReservationEvent con copias defensivas
Actividad 2: Lógica Funcional (Rama: feature/logica-funcional)
Crea una clase utilitaria (ej. ReservationFilters) donde definas como variables (usando Lambdas):
Un Predicate<ReservationEvent> que valide que el precio sea mayor a 0 y que la lista de emails no esté vacía.
Un Consumer<ReservationEvent> que imprima por consola el evento procesado.
Actividad 3: Flujo Reactivo (Rama: feature/api-reactiva)
Crea un controlador REST que exponga el endpoint GET /api/reservations/stream retornando un Flux<ReservationEvent>.
Genera un flujo de 5 reservas en memoria (3 válidas, 2 inválidas) usando Flux.just().
Aplica .filter() usando tu Predicate.
Aplica .doOnNext() usando tu Consumer.
Aplica el operador .defaultIfEmpty() proveyendo una reserva genérica por defecto si el flujo queda vacío tras el filtro.
